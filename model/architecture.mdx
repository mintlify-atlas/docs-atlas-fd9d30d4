---
title: "Neural Network Architecture for Heart Disease Classification"
description: "A 3-hidden-layer Sequential Keras model with a 64→32→16 neuron pattern, chosen to balance expressiveness and generalization on 13 clinical features."
---

The heart disease classifier is built as a Keras `Sequential` model designed for tabular binary classification. It takes 13 clinical features as input and outputs a single probability score indicating the likelihood of heart disease. The architecture was deliberately kept shallow enough to generalize well on only 1,025 samples, while deep enough to capture the non-linear relationships present in the data.

## The `build_model` function

The main model is constructed through a reusable factory function that accepts configurable hyperparameters for activation, optimizer, and dropout rate:

```python
def build_model(activation='relu', optimizer='adam', dropout_rate=0.3):
    model = keras.Sequential([
        layers.Input(shape=(13,)),
        layers.Dense(64, activation=activation),
        layers.Dropout(dropout_rate),
        layers.Dense(32, activation=activation),
        layers.Dropout(dropout_rate),
        layers.Dense(16, activation=activation),
        layers.Dense(1, activation='sigmoid')
    ])
    model.compile(
        optimizer=optimizer,
        loss='binary_crossentropy',
        metrics=['accuracy']
    )
    return model
```

## Layer-by-layer breakdown

| Layer | Neurons | Activation | Notes |
|---|---|---|---|
| Input | 13 | — | One neuron per clinical feature |
| Hidden 1 | 64 | ReLU | Dropout(0.3) after |
| Hidden 2 | 32 | ReLU | Dropout(0.3) after |
| Hidden 3 | 16 | ReLU | No dropout |
| Output | 1 | Sigmoid | Outputs probability 0–1 |

**Total parameters:** 3,521 (13.75 KB) — a deliberately lightweight model for a dataset of 1,025 samples.

**Loss function:** Binary Cross-Entropy  
**Default optimizer:** Adam

## Design decisions

<AccordionGroup>
  <Accordion title="Why 3 hidden layers?">
    Three hidden layers reflect a deliberate bias-variance trade-off for a 13-feature dataset:

    - **Too few layers (e.g., 1):** High bias — the model is too simple to capture non-linear patterns in the clinical data. It underfits.
    - **Too many layers (e.g., 5+):** High variance — the model memorizes training examples and fails to generalize. It overfits, especially on a dataset this small.
    - **Three layers** sits at a reasonable middle ground: expressive enough to learn meaningful patterns, constrained enough to generalize.

    This was validated empirically by also training shallow (1 hidden layer) and deep (5 hidden layers) variants and comparing test accuracy.
  </Accordion>

  <Accordion title="Why the 64→32→16 neuron halving pattern?">
    Each hidden layer is half the size of the previous one. This compression hierarchy serves a specific purpose:

    - **Layer 1 (64 neurons):** Learns broad, high-dimensional representations of the 13 input features.
    - **Layer 2 (32 neurons):** Compresses and refines those representations, discarding noise.
    - **Layer 3 (16 neurons):** Produces a concentrated, low-dimensional encoding that the output layer can classify cleanly.

    This funnel structure mirrors encoder-style design: learn wide, compress narrow, then classify.
  </Accordion>

  <Accordion title="Why Sigmoid on the output layer?">
    The output layer uses a single neuron with a Sigmoid activation. Sigmoid squashes any real-valued input to the range [0, 1], making the output directly interpretable as a probability:

    - Output > 0.5 → predicted class: heart disease present (1)
    - Output ≤ 0.5 → predicted class: no heart disease (0)

    This is the standard choice for binary classification with Binary Cross-Entropy loss.
  </Accordion>

  <Accordion title="Why ReLU in hidden layers?">
    Sigmoid used in hidden layers causes the vanishing gradient problem — as the network deepens, gradients shrink exponentially, making learning very slow or impossible. ReLU (Rectified Linear Unit) passes positive values unchanged and zeroes out negatives, which keeps gradients healthy during backpropagation and enables much faster convergence. The empirical results confirm this: ReLU achieved ~97% test accuracy versus ~83% for Sigmoid in hidden layers.
  </Accordion>
</AccordionGroup>

## Architecture comparison

Three architectures were compared to validate the 3-layer design:

| Architecture | Layers | Neurons | Test Accuracy |
|---|---|---|---|
| Shallow | 1 hidden | 64 | Lower |
| **Main model** | **3 hidden** | **64→32→16** | **~97%** |
| Deep | 5 hidden | 128→64→32→16→8 | Similar or lower |

The shallow model underfits the data. The deep model, despite having more capacity, does not improve accuracy and risks overfitting on a dataset of this size. The 3-layer model is the sweet spot.

The shallow and deep model definitions for reference:

```python
def build_shallow_model():
    model = keras.Sequential([
        layers.Input(shape=(13,)),
        layers.Dense(64, activation='relu'),
        layers.Dropout(0.3),
        layers.Dense(1, activation='sigmoid')
    ])
    model.compile(optimizer='adam', loss='binary_crossentropy', metrics=['accuracy'])
    return model

def build_deep_model():
    model = keras.Sequential([
        layers.Input(shape=(13,)),
        layers.Dense(128, activation='relu'),
        layers.Dropout(0.3),
        layers.Dense(64, activation='relu'),
        layers.Dropout(0.3),
        layers.Dense(32, activation='relu'),
        layers.Dropout(0.3),
        layers.Dense(16, activation='relu'),
        layers.Dropout(0.3),
        layers.Dense(8, activation='relu'),
        layers.Dense(1, activation='sigmoid')
    ])
    model.compile(optimizer='adam', loss='binary_crossentropy', metrics=['accuracy'])
    return model
```
